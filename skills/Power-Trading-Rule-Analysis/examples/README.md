## 使用规则

1. **每个省份单独一个范例章节**
2. **范例基于用户确认"达标了"的报告版本**
3. **后续生成该省报告时，必须严格对照本省范例**
4. **范例内容包括**：章节结构、特色内容、核心参数、政策依据等

## 范例库更新流程

### 入库条件

报告必须满足以下条件才能入库：
- ✅ 用户确认"达标了"
- ✅ 参数项数≥80 项
- ✅ 政策依据≥12 个文件
- ✅ PDF≥30 页
- ✅ 无跨省套用规则
- ✅ 无"待明确"等模糊表述

### 入库步骤

1. **复制报告到范例库**
   ```bash
   cp docs/{省份} 电力交易规则分析报告.md examples/{省份} 范例 V{版本}.md
   ```

2. **更新本索引文件**
   - 添加省份范例条目
   - 填写核心参数和特色

3. **记录到长期记忆**
   - 更新 `long_term_memory.md`
   - 记录入库日期和版本

### 使用范例

生成新省份报告时：
1. 读取对应省份范例（如生成山东报告读山东省范例）
2. 对标参数项数、章节结构、政策依据数量
3. 严禁跨省套用（如山东 D1 曲线不能套用到山西）
