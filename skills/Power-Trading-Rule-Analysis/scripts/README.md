# 报告内容完整性检查脚本

## 用途

自动检查电力交易规则解析报告是否包含所有必需内容，确保报告质量符合 Skill 规范。

## 使用场景

1. **生成报告后**：立即执行检查，确保内容完整
2. **修改报告后**：验证修改是否引入新的缺失
3. **批量检查**：检查多个省份报告的一致性

## 使用方法

### 基础用法

```bash
# 检查单个报告
python3 scripts/check_report_completeness.py <报告文件路径>

# 示例
python3 scripts/check_report_completeness.py docs/山东省电力交易规则解析报告（2026 版）.md
```

### 在 Skill 工作流程中的位置

```
第 1 步：生成 Markdown 报告
         ↓
第 2 步：执行内容检查脚本 ← 这里
         ↓
第 3 步：根据检查结果补充缺失内容
         ↓
第 4 步：再次检查，确保所有项通过
         ↓
第 5 步：转换为 PDF
         ↓
第 6 步：发送给用户
```

### 五 -1. 环保电价参数（5 项，强制固化·不可省略）

**重要说明**：此错误已多次发生，必须固化到检查脚本中！

| 参数 | 要求 | 全国统一标准 |
|------|------|-------------|
| 脱硫电价 | 必须有具体数值或标注"待核实" | 0.015 元/kWh |
| 脱硝电价 | 必须有具体数值或标注"待核实" | 0.010 元/kWh |
| 除尘电价 | 必须有具体数值或标注"待核实" | 0.002 元/kWh |
| 环保电价合计 | 必须计算三项合计 | 0.027 元/kWh |
| 燃煤上网电价 | 必须=基准价 + 环保电价 | 基准价 +0.027 |

**错误案例**（多次发生）：
- ❌ 安徽 V1.0：只写燃煤基准价 0.3844 元/kWh，漏掉环保电价
- ❌ 河南 V1.0：燃煤基准价写"待明确"，未搜索确认
- ❌ 多份报告：附录表格省略脱硫/脱硝/除尘参数项

**正确做法**：
- ✅ 查得到具体数值：填写具体数值（如 0.015 元/kWh）
- ✅ 查不到具体数值：标注"待核实" + 参考值（如 0.015 元/kWh，全国统一）
- ❌ 绝对禁止：省略参数项、写"按规定执行"等废话

**政策依据**：
- 发改价格〔2014〕536 号《燃煤发电机组环保电价及环保设施运行监管办法》
- 全国统一标准：脱硫 0.015 元/kWh、脱硝 0.010 元/kWh、除尘 0.002 元/kWh

## 输出说明

### 通过项（✅）
- 报告显示该项内容完整
- 无需补充

### 警告项（⚠️）
- 可能缺失某项内容
- 建议检查确认
- 不影响报告整体质量

### 问题项（❌）
- 明确缺失某项必需内容
- **必须补充**
- 否则报告不完整

## 修复流程

1. **查看问题项列表**
   ```
   ❌ 问题项 (13):
     ✗ 缺失章节：第八章 辅助服务市场
     ✗ 缺失表格：交易品种总览表
     ...
   ```

2. **对照 Skill 范例补充内容**
   - 读取 Skill 附录 A 范例
   - 找到对应章节的范例内容
   - 按范例模式补充缺失内容

3. **重新执行检查**
   ```bash
   python3 scripts/check_report_completeness.py docs/报告.md
   ```

4. **确认所有问题项已修复**
   - 问题项应为 0
   - 警告项可接受
   - 通过项应≥35 项

### 方案 2：自动化脚本
创建 `generate_and_check.sh`：
```bash
#!/bin/bash
# 生成报告并自动检查

REPORT_PATH="$1"

echo "生成 PDF..."
mdpdf "$REPORT_PATH" --output "${REPORT_PATH%.md}.pdf"

echo "检查内容完整性..."
python3 scripts/check_report_completeness.py "$REPORT_PATH"

if [ $? -eq 0 ]; then
    echo "✅ 检查通过，可以发送"
else
    echo "❌ 检查未通过，需要补充内容"
    exit 1
fi
```

在 Skill 中添加生成后钩子：

```python
# 生成报告后自动调用检查脚本
import subprocess

    ['python3', 'scripts/check_report_completeness.py', report_path],
    capture_output=True,
    text=True
)
if result.returncode != 0:
    print("报告内容不完整，请补充缺失项")
    print(result.stdout)
```

## 常见问题

### Q1: 为什么显示"缺失"但我报告里有？
**A**: 检查脚本使用正则匹配，可能表述方式不同导致未匹配。建议：
- 使用 Skill 范例中的标准表述
- 检查表格标题是否完整
- 确保关键词出现在标题或显眼位置

### Q2: 警告项需要处理吗？
**A**: 警告项不影响报告整体质量，但建议处理以提升完整性。

### Q3: 如何添加新的检查项？
**A**: 编辑 `check_report_completeness.py`，在对应检查方法中添加：
```python
def check_custom_items(self):
    """检查自定义项目"""
    custom_items = [
        ("项目名称", r"正则表达式"),
    ]
    for item_name, pattern in custom_items:
        if re.search(pattern, self.content, re.IGNORECASE):
            self.passed.append(f"✓ {item_name}")
        else:
            self.issues.append(f"✗ {item_name}")
```
