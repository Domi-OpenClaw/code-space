## Tool 定位

独立质量检查 Tool，在报告交付给用户前进行**深度把关**，确保章节标题正确、内容完整、参数齐全、格式规范。

**核心原则**：**深度检查·一票否决制**

> 不能浮于表面！章节标题不对、内容不符合要求的坚决不能出场

**调用时机**：
- ✅ 完整报告生成完成后、交付用户前（强制调用）
- ✅ 关键参数表生成完成后、交付用户前（强制调用）
- ✅ 单章输出生成完成后、交付用户前（强制调用）

**严禁行为**：
- ❌ 只检查章节数量，不检查章节标题
- ❌ 只检查有没有内容，不检查内容质量
- ❌ 浮于表面的检查
- ❌ 未经本 Tool 检查直接交付报告给用户
- ❌ 检查结果不合格仍强行交付

## 检查清单（强制执行·深度检查·一票否决）

### 一、章节标题正确性检查（最高优先级·深度检查）

**完整报告标准章节标题**（必须完全匹配）：
- [ ] **第一章 市场主体概览**（不能是"市场主体规则"等其他标题）
- [ ] **第二章 中长期交易规则**（不能是"中长期交易"等简化标题）
- [ ] **第三章 现货市场规则**（不能是"现货市场"等简化标题）
- [ ] **第四章 零售市场规则**（不能是"零售市场"等简化标题）
- [ ] **第五章 绿电交易规则**（不能是"绿电交易"等简化标题）
- [ ] **第六章 计量结算规则**（不能是"计量、结算与费用"等错误标题）
- [ ] **第七章 偏差考核规则**（不能是"偏差考核与修正"等错误标题）
- [ ] **第八章 辅助服务规则**（不能是"辅助服务市场"等错误标题）
- [ ] **附录 B 关键参数汇总表**（不能缺失）

**一票否决项**：
- 🔴 章节标题与标准不符 → **报告作废，重新生成**
- 🔴 章节标题简化（如"中长期交易"vs"中长期交易规则"） → **报告作废，重新生成**
- 🔴 章节标题错误（如"计量、结算与费用"vs"计量结算规则"） → **报告作废，重新生成**
- 🔴 缺少任何一章 → **报告作废，重新生成**

### 二、章节内容深度检查（最高优先级·深度检查）

**各章节必须包含的核心内容**：

#### 第一章 市场主体概览
- [ ] 市场主体分类（发电企业、电力用户、售电公司、电网企业、交易机构、调度机构）
- [ ] 准入条件（各类主体准入要求）
- [ ] 注册流程（注册步骤、审核时限）
- [ ] 退出机制（强制退出、自愿退出）

**一票否决项**：
- 🔴 缺少市场主体分类 → **报告作废**
- 🔴 缺少准入条件 → **报告作废**

#### 第二章 中长期交易规则
- [ ] 交易品种（9 种：年度、多月、月度、月内等）
- [ ] 交易方式（双边协商、集中竞价、挂牌交易、滚动撮合）
- [ ] 交易日历（具体日期、时间节点）
- [ ] 申报规则（电量、价格、申报段数）
- [ ] 出清机制（边际出清、撮合原则）
- [ ] 签约比例（年度≥80%、月度累计≥90%）

**一票否决项**：
- 🔴 缺少交易品种详解 → **报告作废**
- 🔴 缺少签约比例 → **报告作废**

#### 第三章 现货市场规则
- [ ] 市场时序（日前、实时）
- [ ] 出清机制（SCUC、SCED、RUC/EC 两阶段）
- [ ] 价格形成（节点电价 LMP、统一结算点电价 SMP）
- [ ] 价格限价（上限、下限、负电价）
- [ ] 结算机制（山东 40%+60%、山西中长期 + 现货偏差等）

**一票否决项**：
- 🔴 缺少出清机制详解 → **报告作废**
- 🔴 缺少价格限价 → **报告作废**
- 🔴 山东报告结算机制错误（写"差价结算"） → **报告作废**

#### 第四章 零售市场规则
- [ ] 套餐类型（按该省实际政策分类）
- [ ] 价格公式（每种套餐的价格计算公式）
- [ ] 适用对象（各类套餐适用用户类型）
- [ ] 风险条款（价格风险、偏差风险）
- [ ] 套餐选择建议（按用户类型推荐）

**一票否决项**：
- 🔴 缺少价格公式 → **报告作废**
- 🔴 套餐分类跨省套用 → **报告作废**

#### 第五章 绿电交易规则
- [ ] 绿电定义（风电、光伏等）
- [ ] 交易品种（年度、月度等）
- [ ] 认证机制（绿证、环境权益）
- [ ] 溢价范围（绿电溢价水平）

**一票否决项**：
- 🔴 缺少绿电定义 → **报告作废**

#### 第六章 计量结算规则
- [ ] 计量口径（上网电量、下网电量）
- [ ] 结算流程（结算单出具、资金支付）
- [ ] 费用分摊（辅助服务费用、容量补偿费用）
- [ ] 结算机制（40%+60%、差价结算等）

**一票否决项**：
- 🔴 缺少结算流程 → **报告作废**
- 🔴 缺少费用分摊 → **报告作废**

#### 第七章 偏差考核规则
- [ ] 考核标准（偏差比例、考核价格）
- [ ] 免考核范围（价格联动套餐、新能源等）
- [ ] 修正机制（偏差修正、豁免条件）

**一票否决项**：
- 🔴 缺少考核标准 → **报告作废**
- 🔴 缺少免考核范围 → **报告作废**

#### 第八章 辅助服务规则
- [ ] 市场品种（调频、调峰、备用、无功调节、黑启动）
- [ ] 补偿标准（调频 5-15 元/MW 等）
- [ ] 分摊机制（发电侧分摊、用户侧分摊）

**一票否决项**：
- 🔴 缺少补偿标准 → **报告作废**
- 🔴 缺少分摊机制 → **报告作废**

### 三、参数完整性检查（最高优先级）

**核心参数（6 项必查）**：
- [ ] 燃煤基准价（元/kWh）有具体数值
- [ ] 容量补偿（元/千瓦·年）有具体数值
- [ ] 现货价格上限/下限（元/MWh）有具体数值
- [ ] 中长期签约比例（年度/月度）有具体数值
- [ ] 中长期结算机制有具体公式
- [ ] 偏差免考核范围（±%）有具体数值

**一票否决项**：
- 🔴 6 项核心参数任何一项写"待明确" → **报告作废，重新搜索**
- 🔴 核心参数无政策文号支撑 → **报告作废，补充依据**
- 🔴 参数项数 <60 项（关键参数汇总表） → **报告作废，补充参数**

### 四、政策文件检查

**政策文件数量**：
- [ ] 完整报告 ≥12 个政策文件
- [ ] 关键参数表 ≥12 个政策文件
- [ ] 单章输出 ≥5 个政策文件

**政策文号格式**：
- [ ] 每项参数有政策文号支撑
- [ ] 文号格式正确（如：鲁发改能源〔2026〕160 号）
- [ ] 无"按 XX 号文执行"等模糊表述

**一票否决项**：
- 🔴 政策文件数量 <12 个（完整报告） → **报告作废，补充文件**
- 🔴 编造政策文号 → **报告作废，重新生成**

### 五、数据来源真实性检查（最高优先级）

**数据来源表格**：
- [ ] 政策文件来源明确（官网链接或文件名）
- [ ] 月报/日报来源明确（如无法获取标注"需登录访问"）
- [ ] 核心参数来源标注（搜索确认/官方文件）

**一票否决项**：
- 🔴 捏造月报/日报/公告名称 → **报告作废，重新生成**
- 🔴 编造政策文件文号 → **报告作废，重新生成**
- 🔴 未实际获取的数据标注"已确认" → **报告作废，修正标注**

### 六、格式规范性检查

**Markdown 格式**：
- [ ] 表名置于表格上方
- [ ] 表编号格式正确（表 X-X）
- [ ] 章节编号正确（第一章、第一节、一、（一））
- [ ] 无格式错误（乱码、断行）

**文件大小**：
- [ ] 完整报告 Markdown ≥60KB
- [ ] 完整报告 PDF ≥1.5MB
- [ ] 关键参数表 Markdown ≥5KB

**一票否决项**：
- 🔴 Markdown <60KB（完整报告） → **报告不合格，需补充内容**
- 🔴 PDF <1.5MB（完整报告） → **报告不合格，需补充内容**

### 七、山东 40/60 结算机制专项检查（如适用）

**山东省报告必查**：
- [ ] 结算机制写 40% 固定 +60% 实时联动（非差价结算）
- [ ] 标注政策依据（《山东电力市场规则（试行）》2025 年 12 月修订）
- [ ] 区分 40% 固定部分和 60% 联动部分
- [ ] 偏差电量 100% 按实时节点电价结算
- [ ] 容量补偿独立计算（165 元/千瓦·年）

**一票否决项**：
- 🔴 山东报告写"差价结算" → **报告作废，重新生成**
- 🔴 未标注 2025 年 12 月修订 → **报告作废，补充依据**

### 八、统一架构原则检查

**关键参数汇总表**：
- [ ] 附录 B 调用 `key-parameter-summary` Tool（非嵌入模板）
- [ ] 一张完整表格（严禁拆分成多张小表）
- [ ] 9 大类完整（类别顺序固定）
- [ ] 参数项数 ≥60 项
- [ ] 五列格式：类别 | 参数 | 数值 | 说明 | 政策依据

**一票否决项**：
- 🔴 附录 B 未调用 `key-parameter-summary` Tool → **报告作废，重新生成**
- 🔴 拆分成多张小表 → **报告作废，重新生成**

## 检查流程（强制 6 步·深度检查）

### Step 1: 接收待检查报告

**输入**：
- 报告 Markdown 文件路径
- 报告类型（完整报告/关键参数表/单章输出）
- 省份名称

### Step 2: 章节标题正确性检查（深度）

**操作**：
- 读取报告 Markdown 文件
- **逐字匹配章节标题**（不能简化、不能错误）
- 检查章节顺序（第一章→第二章→...→第八章→附录 B）
- 标记标题错误章节

**一票否决**：
- 章节标题与标准不符 → 终止检查，返回错误
- 章节标题简化 → 终止检查，返回错误

### Step 3: 章节内容深度检查

**操作**：
- 逐章检查核心内容
- 检查各章必须包含的要素
- 标记缺失内容

**一票否决**：
- 缺少核心内容 → 终止检查，返回错误

### Step 4: 参数完整性检查

**操作**：
- 提取关键参数汇总表
- 统计参数项数
- 检查 6 项核心参数
- 检查政策文号

**一票否决**：
- 参数项数 <60 项 → 终止检查，返回错误
- 核心参数写"待明确" → 终止检查，返回错误

### Step 5: 数据来源真实性检查

**操作**：
- 检查数据来源表格
- 核实月报/日报标注
- 检查政策文号格式

**一票否决**：
- 发现捏造数据来源 → 终止检查，返回错误

### Step 6: 生成检查报告

**输出**：
```markdown

**检查时间**：2026-03-21 HH:MM

**省份**：XX 省
**检查结果**：✅ 通过 / 🔴 不通过

### 章节内容深度检查
```
