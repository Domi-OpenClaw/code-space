## BCF Skill知识管理规范

1. **每个BCF Skill执行后必须报结果**（SOUL.md规定）
2. **写入口唯一原则**：只有 `learn.py` 可写 TinyDB，其他系统只读
3. **溯源不可丢**：所有知识点必须带 `archive_url` CDN链接
4. **Knowledge Graph更新**：新增知识点后必须同步更新图谱
5. **文件路径更新**：迁移文件后需同步更新所有引用

## 知识退出机制

```python
# config.py
EVICT_ENABLED = False  # 暂不启用，保留所有知识点
EVICT_MIN_QUALITY = 30  # 低于此分数触发退出审核
EVICT_MAX_AGE_DAYS = 365  # 超过此天数未引用触发退出审核
```

**当前策略**：暂不启用自动退出，所有知识点永久保留。
**原因**：知识价值具有时效性但难以准确量化，手动审核更稳妥。

## 豁免清单（不参与质量评估的条目类型）

以下类型的知识点免于 quality 评分审查，直接入库：

| 类型 | 说明 |
|------|------|
| **政策文件原文** | 发改委/能源局/数据局官方文件 |
| **招投标公告** | 行业关键词 + 公告类型 双重判断 |
| **标准规范** | NDI-TR / TC609 等技术标准原文 |
| **企业官方资料** | 朗新科技等合作伙伴官方材料 |
